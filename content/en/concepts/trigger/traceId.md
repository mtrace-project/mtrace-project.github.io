## Configuration
The TraceId trigger allows you to specify a known **traceId** to run the test on a specific trace that is already present in the observability backend.   

The TraceId trigger is configurable using the `traceId` type and the following arguments:
| Argument  | Description                     | Optional  |
| --------- | ------------------------------- | --------- |
| `traceId` | TraceId to use for the test     | No        |
